## What a Converted Lead Is Worth to Credex

**What Credex sells:** AI infrastructure and credits — managed access to frontier model APIs (OpenAI, Anthropic, Gemini), usage optimization, and spend management at the infrastructure layer. Think: cost-optimized AI gateway with observability and controls for engineering teams.

**Estimated average contract value (ACV):**
- Target customer: 15–80 person company, 8–25 engineers using AI tools
- Monthly API spend at this stage: $2,000–$15,000/month
- Credex takes a management fee or marks up credits; assume 15–20% margin
- Conservative ACV estimate: **$18,000–$36,000/year** (midpoint: $24,000)

**Close rate from warm consultation lead:**
- These are not cold leads. They've already self-identified as having an AI spend problem, completed a structured audit, and explicitly requested a consultation.
- Comparable B2B warm lead close rates in infrastructure SaaS: 20–35%
- Conservative estimate: **25% close rate**

**Therefore: Lead value = $24,000 × 25% = $6,000 per qualified consultation lead**

Even if actual ACV is half the estimate, a converted lead is worth $3,000. That's the floor.

## Conversion Funnel Math

Full funnel for 1,000 visitors:

```
1,000  visitors
  ↓  35% start audit
  350  audit starts
  ↓  65% complete (drop-off on form inputs)
  227  audit completions
  ↓  15% capture email (shown on results page if savings > $500/mo)
   34  email captures
  ↓  30% book consultation
   10  consultations booked
  ↓  25% close rate
    2.5  deals closed
```

**Revenue from 1,000 visitors:**
- 2.5 deals × $24,000 ACV = **$60,000** in pipeline
- At 25% close rate = **$15,000 in recognized revenue**
- Cost to drive 1,000 visitors (community + content, no paid): ~$300–500 time cost

**Blended CAC per closed deal:** ~$150  
**LTV/CAC ratio:** 160x (before any expansion revenue)

## What Would Have to Be True for $1M ARR in 18 Months

**Working backwards:**

| Variable | Number |
|---|---|
| Target ARR | $1,000,000 |
| Average deal size (ACV) | $24,000 |
| **Customers needed** | **42 customers** |
| Months to get there | 18 |
| **New customers per month** | **2.3/month** |
| Close rate from consultation | 25% |
| **Consultations needed per month** | **9–10** |
| Email → consultation rate | 30% |
| **Emails needed per month** | **30** |
| Audit completion → email rate | 15% |
| **Audit completions needed per month** | **200** |
| Audit start → completion rate | 65% |
| **Audit starts needed per month** | **308** |
| Visitor → audit start rate | 35% |
| **Visitors needed per month** | **880** |

**Is 880 visitors/month achievable?** Yes — a single HN front-page post drives that in 24 hours. Sustained community presence, one newsletter feature, and Credex's existing customer base make this very achievable by month 3–4.

**The real constraint isn't traffic — it's consultation capacity.** At 10 consultations/month, Credex needs to have the sales bandwidth to handle those calls and close them within a reasonable cycle. The tool can scale faster than the sales team if it gains traction; that's a good problem.

**What would drive that volume:**
1. One newsletter feature in The Pragmatic Engineer or TLDR Engineering (can add 2,000–5,000 one-time visitors)
2. Credex's existing customer base (100 companies × 60% completion = 60 audits immediately)
3. Sustained SEO from comparison queries ("cursor vs copilot cost") as content ages
4. Word-of-mouth from the results page ("share your audit" feature)

The math works. The question is whether the results page converts at 15%+ — if that rate drops to 5%, you need 5x the traffic. That makes the UI/UX of the results page the highest-leverage variable in the entire model.
